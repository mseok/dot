# Outline Contract

Use this contract to keep the draft narrow and readable.

## Required fields

Every draft should have:

- `Thesis`
- `Intended reader`
- `Source notes`
- `Roadmap`
- `Main sections`
- `Open questions`

## Recommended section order

1. `Thesis`
2. `Why this matters`
3. `Roadmap`
4. `Main argument`
5. `Open questions`
6. `Source notes`
7. `Claim ledger`

## Main section rules

- each section should answer one clear question
- each section should depend on the previous one
- if a section can be removed without harming the argument, remove it

## Claim ledger rules

For each nontrivial claim, record:

- claim text
- source note or notes
- status: `solid`, `partial`, or `speculative`

Keep the claim ledger compact. It is a drafting aid, not a full bibliography.

## Self-contained narrative rule

- the main body must remain understandable without opening any linked note
- if a linked note contains a crucial definition or distinction, restate the
  needed part locally
- reserve wiki links for provenance sections or optional follow-up reading

## Failure signs

Revise the outline if:

- the thesis changes between sections
- the roadmap promises more than the note bundle can support
- the main argument becomes a list of disconnected summaries
- the main body relies on links to other notes to supply necessary context
