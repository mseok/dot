# Style Rules

Use these rules to keep the output in formal Korean report prose.

## Keep

- compact topic sentences
- evaluation-facing structure
- domain terms where they are standard
- explicit mechanisms and constraints
- restrained conclusions

## Avoid

- blog-style narration
- rhetorical repetition
- motivational or promotional language
- unsupported superlatives
- generic AI transitions

## Rewrite Patterns

### 1. Replace conversational enumeration

Before:

```text
후보물질을 고를 때도, 결합 기전을 해석할 때도, 후속 실험을 설계할 때도 ...
```

After:

```text
후보물질 선정, 결합 기전 해석 및 후속 실험 설계를 포함한 초기 신약개발 단계에서는 ...
```

### 2. Replace soft summary rhythm

Before:

```text
이런 점에서 이 모델은 중요한 의미가 있다.
```

After:

```text
이는 해당 모델이 실제 활용 조건을 고려해 설계되었음을 보여준다.
```

### 3. Replace empty contrast

Before:

```text
단순히 정확도만 높은 모델이 아니라 활용성도 높은 모델이다.
```

After:

```text
정확도 외에도 반복 추론 가능성, 활용 경로, 서비스 연계성을 함께 고려하였다.
```

### 4. Replace AI-flavored recap

Before:

```text
다시 말해, 이 모델은 매우 중요한 의미를 가진다.
```

After:

```text
이러한 결과는 해당 설계가 연구용 시연을 넘어 실제 활용 가능성을 함께 고려하고 있음을 시사한다.
```

## Preferred Endings

Prefer endings such as:

- `... 기반을 마련하였다.`
- `... 구조를 갖추었다.`
- `... 가능성을 확보하였다.`
- `... 방향으로 정리되었다.`
- `... 조건을 반영하였다.`
- `... 기준을 맞추었다.`

Use them only when the evidence supports them.

## Words To Limit

Do not overuse these phrases:

- `핵심은`
- `다시 말해`
- `이런 점에서`
- `쉽게 말해`
- `단순히 ... 아니라`
- `~때도, ~때도`
- `매우`, `크게`, `상당히` without evidence

## Tone Check

If a paragraph sounds like:

- a blog post
- a press release
- a generic AI explanation

rewrite it into:

- claim
- reason
- implication for the report section
